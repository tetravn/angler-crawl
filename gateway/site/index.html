API chính — tương thích Firecrawl

Đặt FIRECRAWL_API_URL trỏ vào gốc gateway. Mọi endpoint có cả /v1 và /v2.

POST /v1/scrape POST /v1/crawl POST /v1/batch/scrape POST /v1/map POST /v1/search POST /v1/research POST /v1/extract POST /v1/deep-research POST /v1/deep-research/stream POST /v1/agent POST /v1/agent/stream POST /v1/monitor POST /v1/transcript GET /v1/logs GET /v1/stats

/deep-research/stream & /agent/stream trả text/event-stream (SSE) — token/step trực tiếp. /scrape nhận fallback: "jina"|"firecrawl" (opt-in) khi nguồn nội bộ bị chặn.

/v1/logs và /v1/stats phơi activity log và thống kê chất lượng crawl (tỉ lệ blocked, stub, FlareSolverr…). Khi ANGLER_API_KEY được đặt, hai endpoint này yêu cầu Authorization: Bearer <key>.
